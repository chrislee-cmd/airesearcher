## 1. 처리 활동 인벤토리

각 활동은 GDPR Art. 30(1)(a)–(g) 의 7개 항목을 채웁니다:
(a) 목적 / (b) 데이터 카테고리 / (c) 데이터 주체 / (d) 수령자 / (e) 국가 전송 / (f) 보관 기간 / (g) 보안 조치

### 1.1 사용자 회원가입 + 인증 (signup, login)

| 항목 | 내용 |
|---|---|
| 목적 (a) | 서비스 제공을 위한 계정 생성 + 인증 |
| 법적 근거 | Art. 6(1)(b) 계약 이행 |
| 데이터 카테고리 (b) | Identifier: email, full_name, password hash, OAuth identifier, IP, user-agent |
| 데이터 주체 (c) | 본 서비스 사용자 (Meteor Research 의 직접 고객) |
| 수령자 (d) | Supabase Auth (processor), Vercel (infrastructure), Mixpanel (analytics — 동의 시), Gmail SMTP (확인 메일) |
| 국가 전송 (e) | US (Supabase 리전 미확정 — SEC11 참조), Mixpanel US, Gmail US — DPF 인증 처리자 |
| 보관 기간 (f) | 계정 활성 동안 + 탈퇴 후 7일 (Soft delete window). 이후 hard delete (PR-SEC5). audit_log 의 login_* 은 365일 |
| 보안 조치 (g) | bcrypt password hashing (Supabase), TLS in transit, RLS scoped queries, MFA (계획 — DEBT), 응답 헤더 (PR-SEC3), rate limit (PR-SEC4) |
| 관련 audit event | `login_success`, `login_failure`, `consent_granted`, `consent_revoked` |

### 1.2 음성 / 영상 / 문서 콘텐츠 처리 (transcripts, video, insights, desk research)

| 항목 | 내용 |
|---|---|
| 목적 (a) | 사용자가 업로드한 인터뷰/회의 콘텐츠 → 전사, 분석, 인사이트 추출 |
| 법적 근거 | Art. 6(1)(b) 계약 이행 + Art. 6(1)(a) LLM 처리에 대한 명시 동의 (`llm_processing` consent) |
| 데이터 카테고리 (b) | Sensitive PII: 음성/영상 raw, 전사 텍스트, 인터뷰 chunk, insights quotes (3자 발화 포함 가능), 키워드/리서치 질문 |
| 데이터 주체 (c) | 사용자 본인 + **3자** (인터뷰 respondent — 사용자가 input 으로 제공). 3자에 대한 통지 의무 Art. 14 는 사용자 측 (`/use-policy` §4 명시) — joint controller 책임 가능성 검토 필요 |
| 수령자 (d) | Supabase Storage + Postgres, Deepgram (전사), ElevenLabs (전사 + TTS), OpenAI (chat / Realtime / transcribe), Anthropic (insights / chat / reports), Twelvelabs (video indexing), LiveKit (WebRTC) |
| 국가 전송 (e) | 모두 US (DPF 인증 — 별도 SCC 불필요). 자세한 매핑은 data-flow.md §4 |
| 보관 기간 (f) | 사용자 삭제까지 영구 (`transcript_jobs`, `insights_quotes`, `interview_chunks`, `video_jobs`). 처리자 측: Deepgram default 30일, OpenAI default 30일 (PR-SEC10 으로 ZDR 협의 진행), Anthropic default zero-retention |
| 보안 조치 (g) | RLS (org 단위), private Storage bucket + signed URL, webhook HMAC 검증, prompt injection 방어 (PR-SEC9 XML 래핑), rate limit (org+user 분리), 사용자 삭제 시 cascade delete (PR-SEC5) |
| 관련 audit event | `data_export_requested`, `data_export_completed`, `account_deletion_requested`, `account_deletion_completed`, `rate_limited`, `rls_violation_attempted` |

### 1.3 실시간 통역 / 라이브 세션 (translate)

| 항목 | 내용 |
|---|---|
| 목적 (a) | 발화 → 실시간 번역. host + 익명 viewer 가 share token 으로 join |
| 법적 근거 | Art. 6(1)(b) 계약 이행 (host) + Art. 6(1)(f) 정당한 이익 (익명 viewer — host 의 안내로 참여) |
| 데이터 카테고리 (b) | Sensitive PII: 음성 stream (WebRTC), transcript / translation text (record_enabled=true 시), 세션 metadata |
| 데이터 주체 (c) | host (인증 사용자) + viewer (익명 / 일부 인증) + 발화에 등장하는 3자 |
| 수령자 (d) | LiveKit (WebRTC SFU), OpenAI Realtime (전사+번역), Supabase (text persist) |
| 국가 전송 (e) | LiveKit Cloud (region 설정 미확정), OpenAI US |
| 보관 기간 (f) | `translate_messages` retention cron = 30일 hard delete (PR-SEC5). 비활성 세션은 4시간 후 share_token 자동 무효 |
| 보안 조치 (g) | host JWT-scoped LiveKit token mint, viewer 는 share token 만 (revocable), record_enabled UI 동의, public viewer 의 rate limit |
| 관련 audit event | `rate_limited`, `permission_denied` |

| 항목 | 내용 |
|---|---|
| 목적 (a) | 유료 플랜 결제 처리, 영수증 발행, 세금계산서 발급 |
| 법적 근거 | Art. 6(1)(b) 계약 + Art. 6(1)(c) 법적 의무 (전자상거래법 5년 보존) |
| 데이터 카테고리 (b) | Financial PII: stripe_session_id, customer_id, 영수증, tax_invoice JSONB (사업자번호 / 대표자명 / 사업장 주소) |
| 데이터 주체 (c) | 사용자 본인 (org owner) |
| 수령자 (d) | Stripe, Lemon Squeezy (MoR), 한국 NTS (세금계산서) |

| 보안 조치 (g) | webhook HMAC 검증, RLS (org owner only), `tax_invoice` 평문 (SEC-016 — PR-SEC13 에서 컬럼 암호화 예정 — DEBT) |
| 관련 audit event | `admin_action` (refund / 수동 credit grant 시) |

### 1.8 에러 / 운영 가시성 (Sentry, Vercel logs)

| 항목 | 내용 |
|---|---|
| 목적 (a) | 운영 안정성 — 에러 추적 + 보안 사고 감지 |
| 법적 근거 | Art. 6(1)(f) 정당한 이익 (운영 안전) |
| 데이터 카테고리 (b) | Metadata: stack trace, route, status code, breadcrumb. Identifier: user.id (hashed UUID). **email / 토큰 / body / cookie 는 beforeSend 에서 redact (PR-SEC12)** |
| 데이터 주체 (c) | 에러가 발생한 세션의 사용자 |

| 관련 audit event | (해당 없음 — Sentry 자체는 audit_log 의 outbound) |

| 항목 | 내용 |

## 3. 국가 전송 (Art. 44–49)

- **EU → US**: 2023.07 EU-US Data Privacy Framework (DPF) certified 처리자에게는 SCC 없이 가능. 우리 처리자 list 의 거의 모두가 DPF 인증
- **EU → KR**: 2021.12 한국 adequacy decision — 한국 처리자 (Kakao/Naver) 에게 SCC 없이 가능
- **Meteor → 처리자 DPA**: 서면 DPA 체결 status 는 별도 트래커 (DEBT) 에서 관리
- **사용자 disclosure**: privacy policy v2 (PR-SEC15) 에서 처리자 list 공개 + 처리자별 위치/DPA URL 명시

## 7. 갱신 트리거

ROPA 는 다음 시점에 재검토:
- **신규 처리자 도입** (예: 새 LLM provider 추가)
- **신규 데이터 카테고리 수집** (예: 사용자 위치 / 디바이스 ID)
- **법적 근거 변경** (Art. 6 항목 swap)
- **EU 사용자 비율 20% 초과 시점**
- **외부 audit (CISO / DPO consultant) 권고**

매 검토는 본 문서 신규 `docs/legal/ropa-vN.md` 사본으로 진행, 이전 버전은 `docs/legal/archive/` 로 보존.
